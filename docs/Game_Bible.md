# OPERATION: OVERKILL

## Master Game Bible (v1.0)

This document defines the vision for an original 2D side-scrolling action game inspired by the pacing and destructibility of Broforce while featuring an original world, characters, mechanics and story.

### Core Pillars
- Responsive movement
- Satisfying weapons
- Destructible environments
- Distinct operatives
- Short replayable missions
- Polish over feature bloat

### Story
The Dominion is an AI-backed military empire. The player joins OVERKILL, an elite resistance team fighting impossible missions against overwhelming odds.

### Gameplay
- Run, jump, roll, slide, wall jump, grapple
- Explosive combat
- Environmental destruction
- Bosses with multiple phases
- Mission-based campaign

### Weapons
Pistol, SMG, Shotgun, Sniper, Rocket Launcher, Grenade Launcher, Minigun, Railgun, Laser Rifle, Flamethrower.

Each weapon has unique recoil, effects, sounds, upgrades and animations.

### Operatives
Every operative has:
- Backstory
- Passive
- Signature weapon
- Gadget
- Ultimate ability
- Mastery progression

### Replayability
- Secrets
- Collectibles
- Daily challenges
- New Game+
- Hardcore mode
- Time trials

### Technical (Godot 4)
Folders: Scenes, Scripts, Resources, Audio, Art, UI, Data.
Autoloads: GameManager, SaveManager, AudioManager, InputManager, CameraManager, SceneManager.
Use Resources for data, Signals for communication, object pooling, state machines.

### Roadmap
1. Vertical Slice
2. Core Combat
3. Enemy AI
4. Progression
5. Campaign
6. Bosses
7. Polish
8. Balancing
9. Release

### Game Feel Checklist
Every action should include animation, particles, sound, camera feedback and responsive controls.

### Stretch Goals
- Local co-op
- Replay mode
- Photo mode
- Mission editor
- Steam Workshop support

### Golden Rule
If a feature doesn't make the game more fun within 30 seconds of gameplay, reconsider it.
